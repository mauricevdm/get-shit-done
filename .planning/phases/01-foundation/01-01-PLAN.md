  <name>Task 1: Add worktree registry commands to gsd-tools.cjs</name>
  <files>get-shit-done/bin/gsd-tools.cjs</files>
  <action>
Add the following worktree subcommands to gsd-tools.cjs:

1. `worktree init` — Create `.planning/worktrees/` directory and initialize `registry.json` with schema:
```json
{
  "version": 1,
  "worktrees": {},
  "locks": {}
}
```

2. `worktree add <phase> <path> --branch <branch> --slug <slug>` — Add worktree entry:
```json
{
  "phase_number": "<phase>",
  "phase_name": "<slug>",
  "branch": "<branch>",
  "path": "<absolute-path>",
  "created": "<ISO-8601-timestamp>",
  "status": "active"
}
```
Key in worktrees object is `phase-{N}`.

3. `worktree remove <phase>` — Remove entry from registry (set status to "removed" or delete entirely).

4. `worktree get <phase>` — Return worktree info as JSON. Exit 1 if not found.

5. `worktree list` — Return all worktrees from registry as JSON array.

6. `worktree status` — Merge registry data with `git worktree list --porcelain` output. Show discrepancies (registry says active but path doesn't exist, or git shows worktree not in registry).

Implementation notes:
- Use existing `safeReadFile` and `safeWriteFile` patterns from gsd-tools.cjs
- Registry path: `.planning/worktrees/registry.json`
- All paths stored must be absolute (use `path.resolve()`)
- Follow existing command pattern structure in gsd-tools.cjs
- Add commands to the help text at top of file

<task type="auto">
  <name>Task 2: Add lock registry commands to gsd-tools.cjs</name>
  <files>get-shit-done/bin/gsd-tools.cjs</files>
  <action>
Add the following lock subcommands to gsd-tools.cjs:

1. `lock record <phase>` — Record lock acquisition in registry.json locks section:
```json
{
  "acquired": "<ISO-8601-timestamp>",
  "owner": "<USER env var or 'unknown'>",
  "pid": <process.pid>,
  "hostname": "<os.hostname()>"
}
```
Key is `phase-{N}`.

2. `lock clear <phase>` — Remove lock entry from registry.

3. `lock check <phase>` — Return lock info if exists, exit 1 if not locked. Output JSON with lock metadata.

4. `lock list` — Return all locks from registry as JSON array.

5. `lock stale <phase>` — Check if lock is stale:
   - Read lock info
   - Check if PID still exists (platform-specific: try `process.kill(pid, 0)`)
   - Return JSON: `{ "stale": true/false, "reason": "pid_dead|age_exceeded|active" }`

Implementation notes:
- These commands manage the REGISTRY portion only (not the filesystem lock directories)
- The actual atomic mkdir lock is handled by the shell script
- This is the record-keeping layer for debugging and recovery
- Use `os.hostname()` for hostname field
  </action>
  <verify>
